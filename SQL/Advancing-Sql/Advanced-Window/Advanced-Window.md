## 📖 Overview

Window Functions are one of the most powerful features in SQL, enabling advanced analytical computations while preserving every row in the result set. Unlike aggregate functions that combine rows into a single output, window functions calculate values across a related set of rows and return a result for each row.

They are widely used in business intelligence, reporting, financial analysis, dashboards, and data analytics to solve complex problems with concise and efficient SQL.

# 🪟 What are Window Functions?

A **Window Function** performs calculations across a group of rows that are related to the current row without collapsing the result set.

Unlike aggregate functions:

| Aggregate Functions | Window Functions |
|---------------------|------------------|
| Return one row per group | Return one value per row |
| Reduce the number of rows | Preserve all rows |
| Used with `GROUP BY` | Used with `OVER()` |

---

# ⚙️ Components of a Window Function

Every Window Function consists of three important components.

---

## 1️⃣ Partitioning

`PARTITION BY` divides the result set into independent groups, and each group is processed separately.

### Common Examples

- Employees by Department
- Restaurants by Cuisine
- Students by Class
- Products by Category

---

## 2️⃣ Ordering

`ORDER BY` defines the sequence of rows inside each partition.

Ordering is essential for:

- Rankings
- Running Totals
- Cumulative Calculations
- Moving Averages

---

## 3️⃣ Window Frame

A frame specifies which rows inside a partition participate in the calculation.

Frames are commonly used for:

- Running Totals
- Rolling Averages
- Cumulative Statistics
- Trend Analysis

# 📚 Common Window Functions

## 🏆 Ranking Functions

| Function | Description |
|----------|-------------|
| `ROW_NUMBER()` | Assigns a unique number to every row |
| `RANK()` | Assigns rankings with gaps |
| `DENSE_RANK()` | Assigns rankings without gaps |

---

## 📊 Aggregate Window Functions

| Function | Purpose |
|----------|---------|
| `SUM()` | Running or partition total |
| `AVG()` | Average within a window |
| `COUNT()` | Count rows |
| `MIN()` | Minimum value |
| `MAX()` | Maximum value |

---

## 🔄 Value Functions

| Function | Description |
|----------|-------------|
| `LEAD()` | Returns the next row's value |
| `LAG()` | Returns the previous row's value |
| `FIRST_VALUE()` | Returns the first value in a window |
| `LAST_VALUE()` | Returns the last value in a window |

---

## 📈 Distribution Functions

| Function | Description |
|----------|-------------|
| `NTILE()` | Divides rows into equal groups |
| `PERCENT_RANK()` | Calculates relative ranking |
| `CUME_DIST()` | Calculates cumulative distribution |

# 📝 Generic Syntax

## Basic Window Function

```sql
FUNCTION_NAME(expression)
OVER (
    PARTITION BY partition_column
    ORDER BY ordering_column
)
```

---

## Ranking Function

```sql
RANKING_FUNCTION()
OVER (
    PARTITION BY partition_column
    ORDER BY ordering_column
)
```

---

## Aggregate Window Function

```sql
AGGREGATE_FUNCTION(expression)
OVER (
    PARTITION BY partition_column
    ORDER BY ordering_column
)
```

---

## Value Function

```sql
VALUE_FUNCTION(expression)
OVER (
    PARTITION BY partition_column
    ORDER BY ordering_column
)
```

---

## Window Frame

```sql
FUNCTION_NAME(expression)
OVER (
    PARTITION BY partition_column
    ORDER BY ordering_column
    frame_clause
)
```

# ✅ Best Practices

- Use `PARTITION BY` only when logical grouping is required.
- Always define a meaningful ordering column.
- Choose the appropriate ranking function based on the desired behavior.
- Use CTEs to improve query readability.
- Index frequently used partitioning and ordering columns.
- Avoid unnecessary window calculations on very large datasets.

---

# ⚠️ Common Issues & Troubleshooting

| Issue | Possible Cause |
|-------|----------------|
| Incorrect rankings | Wrong ordering column or order direction |
| Unexpected duplicate rankings | Incorrect ranking function |
| Slow query performance | Missing indexes or large partitions |
| Incorrect cumulative values | Improper window frame definition |
| Unexpected results | Incorrect partitioning logic |
| Missing rows after filtering | Ranking performed after filtering |

# 💡 Key Takeaways

- Window Functions perform calculations without reducing the number of rows.
- The `OVER()` clause defines the processing window.
- `PARTITION BY` creates logical groups.
- `ORDER BY` controls processing order.
- Window frames define calculation boundaries.
- Ranking functions simplify Top-N analysis.
- Aggregate window functions enable advanced reporting.
- Value functions allow comparisons between adjacent rows.
- Window Functions are essential for modern analytical SQL.
